Use Case Name

Create A Post

Use Case Id

UC-1

Priority

1

Primary Business Actor

Member

Description

This use case describes how a member creates a post.

Scope

Member - The main scope is for the member to be able to create a post.

Goal in context

The goal is to create a post with the post visible when anyone searches for it.

Stakeholders and interest

Member : Wants to create a post

User: Wants to view the post when the post is made

 

Preconditions

A user needs to be a member in order to make a post.

Member must be logged in to the system.

Post Conditions

New post should exist, post can be found by searching by the title

Minimal Guarantees

 

Success Guarantees

The member needs fill in all the fields in the post form.

Trigger

Member decides to make a post

Main Success Scenario

1.       Member: Initiate a request to make a post.

2.       System: Displays the make post form.

3.       Member: Enter the details in the correct fields and submits post.

4.       System: Validates post and indicates success.

5.       System: Brings the member to the post.

Extensions

3.a Member: Leaves one or more fields blank.

4.a Member: Clicks the make a post button.

5.a System: Returns Appropriate error message.

6.a System: Returns the cursor to the first invalid field left by the

Member.

 

3.b Member: The member inserts an image that is to large and submits post.

4.b System displays an Appropriate error message.

5.b System removes the image the Member has entered.

 

3.c Member: Clicks cancel post button

4.c System: Displays a confirmation message asking if the member is sure he wants to cancel.

5.c Member: Clicks the cancel button.

5.c System: Displays the main form.

Technology and Data Variations List

none

Frequency of use

As often as the members wishes to post.
